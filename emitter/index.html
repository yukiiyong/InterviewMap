# 微内核 包括核心模块和插件模块

# 优点: 可扩展性，灵活性。功能隔离。自定义逻辑处理。
  • 注册插件时，把播放器实例(事件派发器)注入到插件中。
  • 插件内部会通过事件派发器(EventEmitter)来监听实例的事件

# 轮询: 浏览器每隔一段时间就会向服务器发请求，然后服务器返回最新的数据给客户端

  • 缺点: 浏览器需要不断向服务器发送请求，然而HTTP请求与响应可能会包含比较长的头部，其中有效的数据比较少，所以会浪费较多的带宽资源 (可以用websocket解决，因为websocket只需完成一次握手，浏览器与服务器就可以创建持久性连接，并进行双向数据传输)

# 观察者模式

定义了一种一对多关系，让多个观察者可以同时监听一个主题对象。

当主题对象的状态发生变化时就会通知所有观察者对象，让它们可以自动更新自己。

# 发布-订阅模式

是一种消息范式，消息的发布者(称为发布者)不会将消息发送给特定的接收者(称订阅者)，而是将消息分类，然后将分别发送给不同的订阅者。同样的，订阅者可以表达自己对一个或多个类别感兴趣，只接收感兴趣的消息，不需要了解具体的发布者。

# 对于微内核的核心设计来说，涉及三个关键技术: 插件管理、插件通信和插件连接

由于业务运行过程中，会出现某个业务需要多个插件协作，这就要求 个插件间进行通信。

但由于插件间没有直接联系，只能通过核心系统。因此核心系统 需要提供插件通信机制。
