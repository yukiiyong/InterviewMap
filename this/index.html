# this是什么

this是一个指针，指向调用函数的对象。即this的值只在函数运行时绑定。

# this的绑定规则

  1. 默认绑定
    在不能应用其他绑定规则时使用默认绑定，通常是独立函数使用。 应用了默认绑定的函数，this值在浏览器的非严格模式下为window，在严格模式下，this值为undefined。在node环境下运行函数，this值为global
  2. 显示绑定
    通过call,apply,bind的方式，显式的指定this所指向的对象。call,apply,bind函数的第一个参数为对应函数的this指向的对象。
    call,apply绑定时会执行函数，call后面运行一系列用逗号‘,’分隔的参数作为传入函数的参数，apply允许第一个参数后跟一个数组作为传入函数的参数。
    而bind绑定时不会运行函数，而是返回这个函数
  3. 隐式绑定
    函数的调用是在某个对象上触发的，即调用位置上存在上下文对象。典型形式为 XXX.fun()
    但需要注意的是在对象属性链中只有最后一层会影响到调用位置。
  4. new绑定
    1. 使用new来调用函数(Fn)，会执行已下操作
    2. 创建一个对象obj
    3. 对这个对象执行原型链接，obj.__proto =Fn.prototype
    4. 传入参数，执行构造函数中的代码，同时函数上下文（this）被指定为这个对象（obj）
    5. 若return的值是一个function或基本类型，返回this，若返回的是一个对象，则对象取代整个new出来的结果
var new = function(func) {
  var o = {}
  o.__proto__ = func.prototype
  //var o = Object.create(func.prototype)
  var result = func.call(o)
  if(result && typeof result === 'object' || typeof result === 'function') {
    return result
  } else {
    return o
  }
}

# 绑定优先级（同时应用多种规则时的优先级）

new绑定 > 显式绑定 > 隐式绑定 > 默认绑定

# 绑定例外

如果我们将null或undefined作为this的绑定对象传入call、apply、bind函数，这些值在调用时会被忽略，实际应用默认绑定规则。

# 箭头函数

  1. 箭头函数是ES6新增的特性，它与普通函数的区别在于它没有自己的this，而是继承外层代码块中的this。箭头函数在使用时需注意
  2. 箭头函数没有arguments对象，该对象在函数体内不存在，若要获取参数，使用rest参数代替
  3. 由于箭头函数没有自己的this，所以不能用call、apply、bind这些函数来改变this的指向
  4. 箭头函数内不能使用yield命令，所以箭头函数不能作为Generator函数
  5. 箭头函数不能作为构造函数，即不可以使用new 命令，会抛出一个错误
  6. 箭头函数没有自己的this，继承外层代码块中的this
