# 模块化进程

  • 简单的全局函数（命名冲突，模块关系不明显）
  • namespace模式（对象的属性）外部可以修改内部的变量
  • IIFE（匿名函数自调用）
  • IIFE 闭包（引入增强）
  • CommonJs 主要用于服务端（同步）
  • AMD 浏览器（异步）
  • CMD（浏览器） 异步
  • ESM

# CommonJS

  1. 输入的是输出值的拷贝
  2. 运行时同步加载
  3. 第二次以后加载值从缓存读取
  4. module指代当前模块，是一个对象。它的exports属性是对外的接口
  5. 加载一个模块，实际上是加载一个模块的exports属性
  6. require 读入并执行一个js文件，然后返回文件的exports对象。若没有存在模块，则报错

# AMD

  • 定义：define([依赖]，function() {//... return 模块})
  • 调用：require([依赖]，function(m1, m2) {...})
  • 缺点:代码书写和阅读困难，模块定义的语义不顺畅

# CMD

  • 特点:依赖就近，延迟执行，
  • 缺点:依赖SPM打包，模块加载逻辑偏重
  • 定义：define(function(require，exports, module){//...}

# ESM

  • 输入的值是输出值的引用
  • ESM模块不是对象，它的对外接口是静态定义，在静态解析阶段生成
  • ESM在静态解析阶段确定输入输出接口以及依赖关系
  • export default 为模块指定默认输出，import时可以指定名字(匿名函数)，也可以解构部分引用
