# provide／inject与组件通信

适用于后代组件访问祖先组件

  1. 这对选项需一起使用，以允许一个祖先组件向其所有子孙组件注入一个依赖，不管组件层级有多深，并在其上下游关系成立期间始终有效
  2. 若熟悉React，这与React的上下文特性十分类似
  3. provide这个属性可以指定我们想提供给后代组件的数据或方法，而在任何后代组件中，我们都可以使用inject来接收provide提供的数据或方法
  • Vue本身不会对provide中的变量进行响应式处理，若想inject接收的变量为响应式，那么provide提供的变量需为响应式
  • 将根组件中组件本身注入provide，那么可以在后代组件中任意访问根组件中所有状态，那么根组件成了全局状态的容器

# 为什么官方推荐vuex而不是provide／inject

  1. vuex中的全局状态每次修改都是可以追踪回溯的，而provide／inject中变量修改无法控制（不知道哪个组件修改了状态）
  2. provide／inject破坏了Vue单向数据流的原则（除了.sync）。若有多个后代组件同时依赖于一个祖先组件，那么只要有一个组件修改了状态，所有后代组件都会收到影响，使得数据不可控之余增加了耦合度。

# provide／inject做全局状态管理原则

更推荐用于编写组件

  1. 多人协作时，做好作用域隔离。
  2. 尽量使用一次性数据作全局状态
