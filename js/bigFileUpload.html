# 大文件上传

  1. requestIdleCallback 空闲时间执行函数
    将在浏览器空闲时调用的函数排队（macro task 和ui rendering后）
    requestIdleCallback执行的方法，会传入一个deadline参数，能够知道当前浏览器空闲时剩余的时间
  2. 时间分片计算hash（不卡顿主线程）
    在浏览器空闲时执行workLoop，workLoop是在浏览器空闲时将每一个分了片的chunk通过FileReader读取，添加到spark中
    而在浏览器忙碌时，会把它重新添加到在空闲时执行的函数队列中（requestIdleCallbak）

# 抽样hash

  1. 计算文件md5，判断文件是否存在
    1.1 文件切成2M大小的切片
    1.2 取第一个和最后一个的全部内容，中间部分取首中尾三个地方各2个字节
    1.3 合并之后的内容，计算md5
    1.4 这个hash的结果，就是文件是否存在。（小概率误判，不准是100％）

# 网络请求并发数控制

  • 把异步请求放在一个队列中，设置一个并发数（如3），发起3个请求。然后有请求结束了就发起下一个请求，直到所有请求执行完毕
  • 并发数max ： 发起一个请求max--，结束一个请求max++

# 慢 启动 策略实现

根据当前网络情况，定一个初始大小，动态调整之后的切片大小

  1. chunk加上size值，不过进度条数量不确定，修改createFileChunk，请求加上时间统计
  2. 比如理想是30秒传一个文件
  3. 初始大小定位1M，如第一个（之后也是）上传花了10秒（小于30秒，rate ＝ time／30 ，offset＝ parseInt（offset ／rate），那么下一个区块大小变成了3M
  4. 如果上传花了60秒，那么下一个区块大小变成500KB，以此类推

进度条优化（类似于硬盘扫描） 文件区块和并发数

# 错误处理（并发重试＋报错）

  1. 请求出错catch把任务重新放到队列中
  2. 出错后progress设置为－1，进度条显示为红色
  3. 数组存储每个文件hash请求的重试次数，做累加。
  • 比如[0,1,2]就是第0 个文件出错一次，第二个文件出错2次。
  1. 一个请求出错超过3次直接reject
