# typeof 的原理

  • typeof一般用于判断一个类型，可以判断number，string，boolean，undefined，function，object，symbol 七种类型
  • js在底层存储变量的时候用机器码的低一至三位存储其类型信息
  • 但是由于null的机器码全为0，所以使用typeof检测的时候会认为是对象（object）。解决办法是用object.prototype.toString.call（）这个方法

# 内存泄漏

内存泄漏是指由于错误造成程序未能释放已经不再使用的内存，从而使内存占用过高

解决：可用chrome中的timeline工具进行内存标配，可视化查看内存变化情况，找出异常点

造成原因

  1. 意外的全局变量，浏览器关闭前无法被回收
  2. 定时器未被正确关闭，导致所引用的外部变量无法被正确释放
  3. 事件监听器 没有被正确销毁（低版本浏览器可能会出现）
  4. 闭包 闭包中的变量被全局变量引用，闭包使用完毕后，闭包中的局部变量不能释放
  5. dom引用 dom元素被删除时，内存中的引用未被正确清空

# 什么是＇use strict＇(严格模式)

  1. 无法意外创建全局变量
  2. 会使引起静默失败的赋值操作抛出异常
  3. 试图删除不可删除的变量会抛出异常，之前此操作不会产生任何效果
  4. 要求函数参数名唯一 5.全局作用域下，this的值为undefined
  5. 捕获一些常见的编码错误，并抛出异常
  6. 禁用一些令人困惑或欠佳的功能 8.无法访问function.caller和function.arguments PS: 以不同严格模式编写的脚本合并后会导致问题

# 箭头函数: ES6中代替函数表达式的语法糖

特点:

  1. 没有自己的this，需要从外部作用域获取this(声明时外部作用域的this)
  2. 不能使用call,apply和bind来绑定this
  3. 不能用作构造函数，使用new命令会报错
  4. 不能用arguments属性，若要使用则用rest参数代替
  5. 不能使用yield命令，即箭头函数不能用作Generator函数

原生对象 由ECMAScript规范定义的JavaScript内置对象

宿主对象 运行时环境(浏览器或node)决定的，如XMLHTTPRequest，Windows
