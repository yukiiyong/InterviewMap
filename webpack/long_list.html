# 长列表优化

  1. 懒加载＋分页 （懒加载即 一次只加载可视区域内的条目，滚动到一定距离加载更多条目，追加到列表数据中
  2. 虚拟列表滚动 每次只渲染可视区域内的列表数，当滚动后动态的追加数据并通过顶部padding来撑起整个滚动内容
  • 定位点poll poll.current.getBoundingClientRect().top <= window.innerHeight 即满足追加条件

# 虚拟列表 实现

  1. 实现虚拟列表需要计算出可视区域中的列表项以及顶部和底部的留白的高度。同时需要解决滚动时留白部分出现在可视区域内以及页面抖动，通过给上下预留位置可以解决这个问题
  2. startIndex endIndex 可视区域 真实列表
  3. 绝对定位元素脱离了文档流，相当于在节点只存了token，通过这个token去映射元素。当元素发生变化时，若采取绝对定位，则只对这一块元素进行重绘。。
